# AQNET Homepage Redesign Guide

## 1. Design Positioning

AQNET의 홈페이지는 광고대행사 포트폴리오 사이트가 아니라, AI와 데이터를 기반으로 마케팅 성장을 운영하는 B2B 테크 브랜드로 보여야 합니다.

이번 리디자인의 핵심 방향은 다음과 같습니다.

- 광고 집행보다 `데이터 연결`, `AI 운영`, `성장 운영체계`를 먼저 인식시킨다.
- 네이비, 블랙, 화이트, 블루 포인트 중심으로 정돈해 전문성과 신뢰감을 강화한다.
- 레퍼런스 로고는 많은 광고주를 맡았다는 과시보다 다양한 산업에서 검증된 운영 경험으로 읽히게 한다.
- Madup의 성장 컴퍼니 포지션, Wisebirds의 서비스 구조감, DMC미디어의 B2B 신뢰감을 AQNET 톤에 맞게 결합한다.

## 2. Information Architecture

상단 메뉴는 요구사항에 맞춰 다음 구조로 정리했습니다.

`Home / About / Service / Solution / Works / Insight / Contact`

기존 `Reference` 메뉴는 제거했습니다. 레퍼런스 로고 벽은 사이트에서 중요한 신뢰 장치이지만, 별도 메뉴로 전면에 세우면 광고주 수를 강조하는 대행사 인상이 강해질 수 있어 `Works` 이후의 보조 신뢰 섹션으로 유지하는 방향이 더 적합합니다.

## 3. Section Strategy

### Home

기존 첫 메시지였던 “성과를 만드는 광고 운영”은 광고대행사 인상이 먼저 생길 수 있어, `AI와 데이터로 마케팅 성장을 운영하는 시스템`으로 변경했습니다.

히어로의 우선 메시지는 다음 순서로 읽히게 설계했습니다.

- AQNET 브랜드 로고
- AI & Data Growth Operating Company
- AI와 데이터 기반 성장 운영 시스템
- AQ Growth OS와 검증 사례 CTA

히어로 우측 핵심 역량도 `AI & Data / Performance / Creative` 순서로 재배치해 테크 역량이 먼저 보이게 했습니다.

### Signal & Snapshot

매체명 중심의 채널 스트립은 `Data Pipeline`, `Attribution`, `AI Ops`, `CRM`, `Dashboard`, `Funnel Analytics` 등 운영 역량 키워드로 교체했습니다.

스냅샷 지표는 내부 정리 문구 대신 외부 신뢰와 서비스 구조를 보여주는 표현으로 변경했습니다.

- `70+ Verified Clients`
- `6 Growth Modules`
- `5 Growth Cases`
- `1 Integrated OS`

### About

부정형 메시지인 “광고 운영사가 아니라”는 제거했습니다. 대신 AQNET의 정체성을 긍정형으로 정의했습니다.

`데이터, AI, 크리에이티브, 미디어 운영을 하나의 성장 시스템으로 연결합니다`

운영 원칙은 `진단 / 실행 / 자동화 / 학습`의 4단계로 확장했습니다. 이 구조는 단순 캠페인 실행보다 반복 가능한 운영 체계를 강조합니다.

### Service

서비스 순서는 데이터와 AI가 먼저 보이도록 재배치했습니다.

1. Data Analytics
2. AI Marketing Solution
3. Performance Marketing
4. Creative & Contents
5. CRM / Funnel Consulting
6. Commerce Growth

AI Marketing Solution은 강조 카드로 유지하되, 표현을 `자동 리포팅`, `이상 신호 감지`, `소재/예산 조정 우선순위 추천`, `운영 지식 자산화`처럼 제품형 언어에 가깝게 보강했습니다.

### Solution

`AQ Growth OS`는 페이지의 핵심 브랜드 자산으로 강화했습니다.

운영 루프는 다음 구조로 정리했습니다.

`Collect / Analyze / Decide / Execute / Learn`

기존의 장식형 성과 수치처럼 보일 수 있던 패널은 `AQ Growth OS Console`과 `LIVE` 상태로 변경했습니다. 근거 없는 수치 강조를 줄이고, 데이터 레이어와 다음 액션을 관리하는 운영 콘솔처럼 읽히게 했습니다.

### Works

Works 제목은 `데이터로 검증한 성장 사례와 운영 레퍼런스`로 변경했습니다.

월예산과 매출 수치는 유지하되, 각 카드 설명에 채널 구조, 구매 여정, 콘텐츠 메시지, 퍼널 신호 등 접근 방식을 함께 넣었습니다. 이를 통해 단순 광고 성과표가 아니라 데이터 기반 성장 사례로 보이게 했습니다.

### Client Network

레퍼런스 로고 벽은 유지했습니다. 다만 노란 priority 테두리는 팔레트와 맞지 않아 블루 계열의 얇은 강조로 변경했습니다.

이 섹션의 역할은 “많은 광고주 보유”가 아니라, 다양한 산업군에서 축적한 운영 경험을 보여주는 신뢰 장치입니다.

### Insight

기존 Insight 항목은 모두 Contact로 연결되어 placeholder처럼 보일 수 있었습니다. 클릭 링크 대신 프리뷰형 article 구조로 변경해 실제 리포트/자료가 준비되기 전에도 어색하지 않게 정리했습니다.

### Contact

문의 문구는 `광고 계정` 중심에서 `광고, CRM, 커머스, 고객 행동 데이터` 중심으로 확장했습니다. 실제 운영 시에는 mailto보다 서버 폼, 개인정보 문구, 스팸 방지 구성이 추가되면 좋습니다.

## 4. Visual System

### Palette

라임과 노란색 강조는 제거했습니다. 전체 톤은 아래 계열로 정리합니다.

- Ink Black: `#05070d`
- Deep Navy: `#0a1020`, `#0d1b32`
- Primary Blue: `#2276ff`
- Cyan Signal: `#2ed3ff`
- Soft Signal Blue: `#7ab7ff`
- White / Fog: `#ffffff`, `#f4f7fb`

### Typography

사이트 전체 폰트는 `Pretendard`를 기준으로 사용합니다. 영문과 한글 모두 같은 산세리프 톤으로 통일해 B2B 테크 브랜드의 정돈된 인상을 유지합니다.

CSS 전역 폰트 스택은 다음 기준을 따릅니다.

`"Pretendard", "Pretendard Variable", -apple-system, BlinkMacSystemFont, "Segoe UI", "Noto Sans KR", Arial, sans-serif`

프로젝트 안에 별도 폰트 파일은 포함하지 않았으므로, 배포 환경에서 Pretendard가 없을 경우 시스템 폰트로 fallback됩니다. 완전한 브랜드 일관성이 필요하면 추후 Pretendard 웹폰트 파일을 로컬 assets로 포함하는 방식을 권장합니다.

### Motion

화려한 캠페인 랜딩처럼 보이지 않도록 카드 hover 움직임을 줄였습니다. 애니메이션은 히어로의 데이터 네트워크와 핵심 CTA 정도에만 제한하는 방향이 적합합니다.

### Logo

헤더, 히어로, 푸터는 기존 생성형 PNG 로고를 기반으로 배경을 제거한 `assets/logos/aqnet-logo-header.png`를 사용합니다. 단순 색상 삭제가 아니라 원본의 어두운 렌더 배경과 흰 외곽 여백을 각각 분리해 마스크 처리하고, 가장자리는 부드러운 알파 매팅으로 정리했습니다.

## 5. Implementation Notes

수정된 주요 파일은 다음과 같습니다.

- `index.html`: 정보 구조, 섹션 카피, 서비스 순서, 솔루션 루프, Insight 구조 변경
- `styles.css`: Pretendard 전역 폰트, 팔레트, 레이아웃, 카드 밀도, Snapshot, Reference, Insight 스타일 변경
- `script.js`: 히어로 데이터 스트림 색상을 블루/시안 계열로 조정
- `assets/logos/aqnet-logo-header.png`: 웹용 투명 배경 PNG 로고 락업

## 6. Next Recommendations

향후 실제 운영 단계에서는 다음 보완을 권장합니다.

- Works 수치의 기준 기간과 측정 범위 명시
- Insight 상세 페이지 또는 다운로드 자료 제작
- Contact 폼의 실제 제출 처리, 개인정보 처리 문구, 스팸 방지 추가
- AQ Growth OS를 회사소개서에서도 동일한 핵심 프레임으로 사용

## 7. Redesign v2 — 레퍼런스 반영 + 검토 개선

매드업·와이즈버즈·DMC미디어·비에이티·드리븐 등 1순위 레퍼런스 분석과 현행 사이트 정밀 검토를 종합해 다음을 반영했습니다.

### 메시지 / 히어로

- 히어로 H1을 **이미지 로고 → 실제 텍스트 H1**으로 교체했습니다. 레퍼런스 공통 문법인 “정체성 선언 한 줄 + 영문 서브카피” 구조를 채택해, 클라이언트가 제시한 메인 카피 방향(`퍼포먼스 마케팅을 넘어, 성장 구조를 설계합니다`)을 헤드라인으로 사용했습니다. 폰트 미로드/이미지 깨짐에도 H1이 보이고 접근성·SEO가 개선됩니다.
- `설계`에 블루→시안 그라데이션 포인트를 주어 데이터·테크 톤을 유지하면서 시선을 집중시켰습니다.

### 브랜드 로고

- 헤더·푸터의 다크 배경용 로고를 교체했습니다. 기존 `aqnet-logo-header.png`는 워드마크에 유리질 렌더 아티팩트가, `...-mask-preview.png`는 검은 배경(알파 없음)이 있었습니다.
- 그레이스케일 마스크의 명도를 알파 채널로 변환해 **투명 배경의 깨끗한 화이트 락업** `aqnet-logo-wordmark-white.png`를 생성, 헤더/푸터에 적용했습니다.

### 신뢰 / 데이터 표현

- 채널 스트립을 운영 키워드에서 **공식 운영 채널 & 데이터 스택**(Google Ads·Meta·YouTube·TikTok·Naver GFA·Kakao Moment·GA4·Appsflyer·Airbridge·Amplitude·BigQuery·CRM)으로 교체해 “공식대행 + 데이터 기반”을 시각화했습니다. (드리븐·비에이티 참고)
- 스냅샷 지표는 검증 주체 불명의 `Verified`를 제거하고 `운영 브랜드 / Growth Modules / 대표 성장 사례 / Integrated Growth OS`로 정리, **스크롤 진입 시 카운트업** 모션을 추가했습니다. (더플레이스·와이즈버즈 참고)
- Works 지표 라벨을 `월 예산 / 광고 매출`로 통일하고 측정 기준 주석을 추가했습니다.

### 인터랙션 / 모션

- 주요 섹션에 **스크롤 진입 리빌**(IntersectionObserver)을 추가하되 과하지 않게 제한했습니다.
- Solution 콘솔의 거대한 `LIVE` 텍스트를 **펄스 점이 있는 pill 뱃지**로 정돈했습니다.
- 무한 마퀴 루프 이음새(매직넘버 `-48%`)를 `-50%`로 정렬했습니다.

### 정보구조 / 접근성 / 성능

- `scroll-padding-top` + `scroll-margin-top`으로 fixed 헤더에 앵커 제목이 가리던 문제를 해결했습니다.
- 스킵 링크, 스크롤 전 헤더 가독성 스크림, OG/Twitter/JSON-LD 메타를 추가했습니다.
- 레퍼런스 로고 70장에 `loading="lazy"` + 고정 치수(CLS 방지)를 적용하고, 의미 없이 반복되던 alt를 제거(`alt=""`)한 뒤 컨테이너에 `role="img"`로 단일 레이블을 부여했습니다.
- Works 필터에 `aria-pressed` + 라이브 영역 결과 안내를 추가했습니다.
- 모바일 메뉴에 ESC·외부 클릭·데스크탑 복귀 닫힘과 `inert` 처리를 추가했습니다.
- Contact 폼에 개인정보 수집·이용 **동의 체크박스**와 `role="status"` 안내를 추가했습니다.
- `prefers-reduced-motion`에서 히어로 캔버스를 1회 렌더 후 정지(불필요한 rAF 루프 제거)하고, 리빌·카운트업도 즉시 표시되도록 했습니다.
- 미사용 `.brand-mark` / `.brand-text` CSS와 전역 변수 null 가드 부재를 정리했습니다.

### 신규/변경 자산

- `assets/logos/aqnet-logo-wordmark-white.png`: 헤더·푸터용 투명 배경 화이트 락업 (신규)

## 8. Redesign v3 — 데이터·프로덕트 테크 프리미엄 (비주얼 디벨롭)

v2까지가 구조·카피·접근성 정비였다면, v3는 시각적 디자인 언어 자체를 끌어올린 본격 리디자인입니다. 무드는 매드업/DMC급 "데이터·프로덕트 테크 프리미엄"으로, 네이비/블랙/화이트/블루 톤은 유지하되 깊이·레이어·데이터 모티프를 더했습니다. 구조 재편을 허용했습니다.

### 디자인 시스템 토큰 (styles.css 전면 재작성)

- **Surface 4단계**(`--surface-0~3` + glass), **Border 3단계**, **Text 4단계**, **Elevation 4단계**를 도입했습니다. 핵심은 다크 카드에 `inset 상단 1px 하이라이트`(`--e-dark-1`)를 일괄 적용해 평면 표면을 입체로 만든 것입니다.
- **Radius 단일 8px → 5단계**(`--r-xs/sm/md/lg/pill`)로 위계를 부여했습니다.
- **시그니처 그라데이션·글로우**(`--grad-brand/action/bar`, `--glow-blue/cyan`, `--mesh-cyan/blue`)를 토큰화하고 사용처를 고정해 남발을 막았습니다.
- **타이포 스케일**: eyebrow를 `letter-spacing 0.14em + uppercase`로, 숫자에 `tabular-nums`를 적용했습니다. 라이트 본문 색을 `#3a4861`로 진하게 조정해 대비를 보강했습니다.
- **배경 모티프 유틸**(`.bg-grid/.bg-dots/.bg-mesh`)을 정의했습니다.

### 섹션 구조·비주얼 재편 (index.html)

- **Hero 우측 → 라이브 콘솔 카드 스택**: 글래스 카드에 스파크라인(인라인 SVG) + KPI 3행(Unified Data / Blended ROAS 카운트업 / Next Action) + LIVE 펄스 + 뒤에 겹친 스택 카드. 단순 메트릭 3종을 "살아있는 대시보드"로 교체했습니다. 하단에 Scroll 큐 추가.
- **About 원칙 → 세로 타임라인**: 2×2 그리드를 번호 노드 + 연결선 + hover 강조의 운영 타임라인으로 재구성. statement 박스에 브랜드 삼각형 워터마크(SVG)와 도트 텍스처.
- **Service 카드 격상**: 카드별 **인라인 SVG 아이콘 칩** + 상단 액센트 바(hover fill) + hover 깊이. AI Marketing Solution은 다크 그리드 텍스처 + `FLAGSHIP` pill로 1순위 강조.
- **Solution을 다크 시그니처 섹션으로 승격**: AQ Growth OS 운영 루프를 아이콘 노드 + flow 레일로 만들고, **3초 간격으로 active 노드가 순환**(콘솔 패널 상태 셀과 동기화). 콘솔 차트는 진입 시 막대가 stagger로 차오릅니다.

## 12. Scroll Experience v5 — 스크롤 인터랙션 강화

전수 감사(요구사항 전 항목 충족 확인) 후 "스크롤하면 살아 움직이는" 경험을 추가했습니다. 전부 바닐라 rAF/IO, transform·opacity·filter만 사용, 레이아웃 읽기는 load/resize에서만.

### 신규 스크롤 인터랙션
- **히어로 스크럽**: 스크롤 이탈 시 카피가 translateY(-46px, 모바일 -22px)·scale 0.97·opacity 0.15로 침강 — 콘솔 카드(-18px)와 시차를 만들어 레이어 깊이감. `--hs` 변수(기존 heroProgress 재활용).
- **헤딩 라인 마스크 리빌**: `.section-heading h2`를 JS가 `<br>` 단위로 라인 래핑(노드 이동 방식이라 `.grad`/`.nowrap` 보존, SEO 무영향) → 라인이 아래에서 90ms 간격으로 차오름. 히어로 h1은 제외.
- **카드 stagger 재구축**: transition 기반 → **keyframe(cardIn: translateY+scale 0.965+blur 5px→0)** 전환. 기존 stagger의 transition-delay가 카드 hover 반응을 최대 350ms 지연시키던 **버그 수정**(감사 발견). 모바일은 blur 없는 cardInLite.
- **C·O·R·E 이니셜 스크럽**: 섹션 통과 진행도(`--fwp`)에 따라 고스트 이니셜이 카드별 다른 계수(1/1.4/0.75/1.2)로 부상.
- **채널띠 스크롤 가속**: 죽은 코드였던 scrollVelocity를 소비 — WAAPI `updatePlaybackRate`로 1→최대 3.5배 가속 후 프레임당 5% 감쇠. 미지원 브라우저는 조용히 기본 속도.
- **Solution 루프 스크롤 스크럽**: 섹션 진행도를 5단계 양자화해 스크롤로 루프를 "문지르고", 멈추면 4초 후 기존 3초 자동 순환이 현재 노드에서 재개.
- **로고월 타일 stagger**(인덱스×16ms, 상한 캡 34), **스파크라인 드로우-온**(pathLength=1, 로드 후 1.4s 드로잉).

### 감사 수정 사항
- scrollspy 사각지대: framework→About, reference→Works 프록시 매핑.
- 캔버스 포인터 rect: 스크롤마다 gBCR 호출 제거 → 문서 기준 top 캐시 + 산술 계산.
- `--text-d-4` 0.34→0.45 (다크 소형 텍스트 대비 개선), skip-link `#main` 직행, `.button-primary:hover` 중복 규칙 병합, 리빌 완료 후 `will-change: auto`.
- 미해결 백로그 재확인: **개인정보처리방침 페이지(우선순위 높음)**, footer 실데이터.

## 15. v9 — 히어로 exit 핸드오프 "Signal Convergence" (데이터 스트림 바통터치)

클라이언트 피드백("콘솔을 가운데 보여주는 느낌에서 끝난다 — 기하학적 애니메이션이 다음 섹션으로 넘어가는 효과")에 따라, 기획·개발 2관점 서브에이전트 스펙 후 히어로 핀 후반부에 exit 위상을 신설했습니다.

### 컨셉
"흩어진 신호 → 하나의 시스템"이라는 히어로 서사를 스크롤로 완결: 콘솔 클로즈업 정점 이후 배경 파티클 네트워크가 **수평 스트림 라인(레일)으로 수렴**하고, 무대가 반 발 물러나며 어두워지는 동안 **Signal Strip이 무대 위로 슬라이드 인**해 레일과 바통터치. 스트립 마퀴의 좌향 흐름·스크롤 가속(stripBoost)이 운동 연속성을 이어받음.

### 구조 (트랙 260svh, 위상 분리)
- `.hero` 200→260svh. `HERO_IN_END=0.625`로 위상 분리 — 기존 IN 안무(카피 침강→콘솔 센터→KPI 점등→브리지)는 `pi=remap(p,0,0.625)`에 이전해 **물리적 스크롤 거리 불변**. 뒤 60svh가 exit 전용.
- `.signal-strip{margin-top:-60svh}`(데스크탑) — 트랙 연장분과 정확히 상쇄, **페이지 총 길이 불변**. 후행 섹션(strip·snapshot·about)이 DOM 순서 페인팅으로 핀 무대 위에 그려짐(z-index 미지정, `position:relative`만).
- exit 비트: `--hx`(0.64–0.98, easeInOutCubic) 무대 translateY -4vh·scale 0.955 + `::after` 스크림 opacity 0.45(컴포지터 전용, filter 금지) / `--hrl`(0.70–0.88) 레일 scaleX 드로우 / 캔버스 수렴 `convergeK`(0.66–0.90).
- **레일**: `.hero-rail` top 43.7% — p=1 시점 Signal Strip border-top과 만나도록 무대 transform(translate -4vh·scale 0.955, origin center)을 역산한 값. 실측 정합 델타 0.1px. z-index -1(콘솔 뒤 평면, 양옆으로 드러남).
- **캔버스 수렴**: 파티클 y를 레일 밴드(viewH 43.7%)로 lerp + 마퀴 방향 좌향 드리프트(70px·심도별). 연결선 판정을 이방성(세로 감쇠·가로 증폭)으로 전환해 체인/스트림化. 전부 p의 순수 함수 — 역스크롤 자연 복원, 상태 누적 없음. 페이드는 "사라짐"이 아니라 "모임"이 보이도록 완만화(0.6배) 후 오버랩 종반(0.88–0.99)에 최종 페이드, alpha<0.02면 드로우 패스 스킵.
- 모든 exit 커브 종점 p≤0.98 — sticky 해제 프레임에 이미 정지 상태(덜컥임 방지).

### 가드
- `will-change:transform`은 `.hero.exiting`(p 0.55~1)에만(100svh 레이어 상시 승격 방지). 수렴 중 포인터 리플 무시(convergeK>0.2). resizeCanvas 크기 판독을 gBCR→offsetWidth/Height(transform 독립). lazy 이미지 로드 후 오프셋 재측정(`load`시 measureSections).
- 모바일(≤1024): 전부 미적용 — 트랙 연장·마진·레일·transform 모두 데스크탑 미디어쿼리 내부. 기존 폴백 경로 그대로.
